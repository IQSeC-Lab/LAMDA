Abstract

Machine learning-based malware detection systems are widely used for their ability to identify previously unseen threats. However, their effectiveness diminishes over time due to model degradation. As malware evolves, these models suffer from concept drift, where shifts in data distribution gradually lead to a decline in detection accuracy. To address this issue, it is essential to train models on datasets covering longer periods, as this is essential for revealing long-term trends and patterns. In this paper, we present a novel dataset designed to analyze the effects of concept drift on Android malware detection. Our dataset spans from 2013 to 2025 (excluding 2015) and includes both malware and benign samples, totaling 1,032,497 entries. By highlighting the performance degradation over time, we emphasize the need for datasets that reflect evolving threats and propose methods to address concept drift.

Dataset Preparation

  • Dataset Construction: Over 1.03 million APKs (2013–2025, excluding 2015) were collected from the AndroZoo repository and labeled based on VirusTotal scan results: benign (vt_detection = 0), malware (vt_detection ≥ 4), and uncertain (vt_detection ∈ [1, 3], discarded). Samples were organized into [year]/malware/ and [year]/benign/ directories.
  • Static Feature Extraction (Drebin-inspired): APKs were decompiled with Apktool to extract Drebin-like static features from both the manifest and smali code.
  • Vectorization & Preprocessing: Features were transformed into sparse binary vectors using a bag-of-tokens approach with a global vocabulary from 2013. Low-variance features (var < 0.001) were removed to improve relevance and reduce dimensionality.
  • Storage & Family Labeling: Processed data was saved as .npz files (monthly/yearly), each containing features (X), labels (y), SHA256 hashes, and AVClass2-generated malware family names — enabling both binary and multi-class classification tasks.

Key Contributions

  • A large-scale Android malware benchmark comprising over 1 million APKs across 1,430 unique families, spanning 2013–2025 (excluding 2015), with additional singleton samples. It is the most temporally comprehensive dataset for Android malware analysis to date.
  • Built on execution-free static features—API calls, permissions, and manifest components—extracted using a standardized schema, enabling scalable, reproducible, and temporally aligned drift analysis.
  • Longitudinal evaluations under temporal drift using structured splits, showing that malware classifiers degrade significantly over time and that family-level behavior evolves in measurable, feature-space terms.
  • Facilitates explanation-driven drift analysis through SHAP-based attributions, supporting investigations into how feature importance shifts as malware evolves.
